APSIM Outlook User Manual

 

Introduction

 

APSIM Outlook is an easy-to-use graphics package that allows users to quickly process APSIM Output files and produce time series graphs, difference graphs, xy scatter graphs, probability graphs, frequency graphs and box plots.

 

This manual will guide you through the process of using APSIM Outlook including the steps necessary for producing a wide range of outputs from APSIM Output files.

 

Starting APSIM Outlook

 

There are a number of ways to start APSIM Outlook. The simplest is to right click on a set of APSIM output files in Windows Explorer and select APSIM Outlook. Alternatively goto View Output | Graph using APSIM Outlook in APSFront and select several output files. These options will then automatically  take you to the Simulation Selection Screen.
 

Simulation Selection Screen

 

This screen shows a single simulation selected called: 160-BlighLee N38_N150. It also show that no economic analysis is to be performed. By default only 1 simulation is selected. To select all simulations click on the sim= line.

You then get the option to tick all your simulations. Click the Apply to current scenario option at the bottom to select the ticked simulations. The Apply to all scenarios applies the ticked simulations to those that have already been selected previously, giving a full permutation of simulations. This latter option really only applies with Whopper Cropper style titles. The recommendation is to always use the Apply to current scenario option. You then end up with all simulations on the Choose Scenario form.

Clicking ok now will take you to the main ApsimOutlook window, ready for graphing.

 

The main Window

 

 

The main window is where the simulations can be graphed. The lower toolbar contains options for the different graph types and data analysis tools. The scenario window on the right, shows the currently selected simulations. It can be closed at any time to free up window space.

 Pie Charts

 

Pie… charts enable you to compare the distributions of two sets of data.  This is especially useful for comparing the distribution of a particular SOI phase against that of the entire 100 year simulation.

·          Select the number of pie slices you wish to view.  For example, if you select three pie slices, Whopper Cropper ranks the data and divides it into terciles.  This means that 33% of the data will lie in each pie slice.

Probability Charts
 

Whopper Cropper can also produce cumulative probability distributions (upward sloping), or probability of exceedence graphs (downward sloping) under Charts|Probability menu. 

 

Choose the data you wish to analyse, and the type of chart.

 

Probability of exceedence graphs show the probability of a simulation exceeding a particular value.  The slope is downwards because there is a high probability that yield will exceed low values, and a low probability that yield will exceed high values.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Cumulative probability graphs show the probability that a simulation will be less than a particular level.  They slope upwards because there is a low probability of yield being less than low values and a high probability of yield being less than high values.

 

Summary Tables 


Summary tables provide basic statistics describing simulation outputs in tabular, rather than graphical, form.  This is particularly useful for interpreting some of Whopper Cropper's graphical outputs.  The selection screen is similar to those for charts.

 

The summary statistics include the mean, median, count, minimum and maximum, and deciles.  Drag the window open to view the data (as indicated by the arrow).

Rotation Analyses

 

APSIM Outlook includes an add-in, that when turned on, will process a rotation set of APSIM output files into a single merged file that can be charted using the normal Outlook charting technology.  There are a few requirements on the output files to make this work correctly.

 

APSIM Output File Requirements.

 

  • Each output file fed into Outlook must be offset a year from the previous file.

  • Each output file must have a year or sow_year or harvest_year column.

  • Each crop variable in the file must have the crop name followed by an underscore followed by the variable name e.g. nw_yield.  The list of crop names that Outlook recognises is found in the ApsimOutlook\RotationAddIn\RotationAddIn.ini file.  Currently they are:

      ckp – chickpea

      sor – sorghum

      sorg –sorghum

      lup – lupin

      can – canola

      sun – sunflower

      wht – wheat

      syb – soybean

      pnt – peanut

      oz – ozcot

      nw – nwheat

      nyb – navybean

      mgb – mungbean

      mz – maize

      fab – fababean

      cwp – cowpea

      wd or wd1 or wd2 or wd3 – weeds

      luc - lucerne


 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Rotation Analysis in Outlook

APSIM Outlook looks through all crop files, for each year in a given file it totals the values for a column and then averages this total across all files.  For some columns, it doesn’t make sense to total values for a given year in a given file (e.g. year or pstage).  These variables can be listed in the .ini file and they will be averaged.

            e.g.

            [fields]

            averaged_fields = *year

 

This will average, within a year, any column that has a ‘year’ substring it its name.  If the asterisk is omitted, the column name must match exactly.  This average will then be averaged across all files as normal.

 

For crops that weren’t sown in a particular year, the zeros don’t count towards the average.  A crop is sown if:

       there is a least 1 non zero value in the fields for the crop OR

       the crop_fail field (if it exists) has a 'yes' in it.

 

In addition, the (num files – 1) years at the beginning and end of each file are removed i.e. Outlook only keeps years that are in all files fed to it.
