Simulation Failure

The First Step - Determine the type of failure

There are two main types of failures we need to discern for this first step - Handled and Unhandled Exceptions.

Unhandled Exceptions

A fatal error that is not appropriately dealt with by the program software implementation is called an unhandled exception.  In this case a compilor or operating system error message is given on the screen and the program ceases to operate.   These errors are usually caused by invalid numeric data processing, incorrect memory management or other programming errors.  

Handled Exceptions

It is more preferrable for the software to detect and react to the occurrence of an exception.  Most software tests for possible errors and will fix the error or alert the user to the problem.  APSIM provides both fatal and warning errors to the user.   In the case of a failed simulation, the occurence of a fatal error message in the summary file identifies a handled exception.
