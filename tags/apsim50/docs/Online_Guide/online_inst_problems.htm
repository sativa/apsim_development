Installation Problems
 

  1. When I run the APSIM installation program, I get an error message that says I must have a valid APSIM password. I type in the key given to me, but it fails to accept it.

    2. 1. Be sure that you have typed the password correctly.
    2. The password changes with each release. If you are trying to install a new version with an older key, you will get this message.

    If you have need a new password, or have misplaced your old one, please contact the Help Desk to receive another key to complete a new installation.

  2. I used to have APSuite version 1.4 or lower and now after installing APSuite 1.5, APSFront produces and error message and crashes when I try to start the program.

    3.

    In Windows Explorer, look in the \APSWORK\CONFIG directory. Open the file called APSFRONT.INI. The file will have various lines that reference directories where the APSfront templates and met files are drawn from. Firstly make sure the directories are pointing to the correct relative paths. Secondly, you will notice that the start of the directory reference will have %HOMEDIR or %WORKDIR . For version 1.5 this should now be %APSWORK. The previous versions were are now obsolete and require the newer reference.

  3. I am upgrading from version 1.3. Are there any issues I need to be aware of?

    4.

    (1.6+) Yes. In your AUTOEXEC.BAT file in your computer’s root directory, there will be a reference to an \APSIM directory in the SET PATH statement. When you upgrade to version 1.5 there will be two new references in the SET PATH statement. These are \APSUITE\SHARED and \APSUITE\3RDPARTY. To ensure that APSIM runs correctly you must remove the \APSIM reference from the SET PATH statement in the AUTOEXEC.BAT file.

    (3.0+). Since the release of Apsim 3.0, a change tool will modify your control/parameter files to the current version. Make sure you keep backups to refer to.
