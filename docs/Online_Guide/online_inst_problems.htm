Installation Problems
 

  1. When I run the APSUITE installation program, I get an error message that says I must have a valid APSIM licence. I thought I had one.

    2. Two things may have happened.

    One, you need to install the APSIM Licence Key before you install the APSUITE software. The Help Desk usually assigns an APSIM licence key to you when you are licenced to use APSIM. Contact the APSIM Help Desk to obtain a licence if you have not already done so.

    Two. You have uninstalled a previous version of APSIM. This action will remove APSIM including your key from your computer. Reinstall your licence key.  If you have misplaced your key please contact the Help Desk to receive another key to complete a new installation.

  2. I have asked for permission to use a module in APSIM and have received an updated key. However after I install the key I can’t use the new module.

    3. To get proper access to the module that has been added to your key, you will have to uninstall the current version of APSUITE from your computer, install the key and then re-install APSUITE to your hard disk. The installation process reads the information in your key and then installs the relevant modules. Therefore it is important that you uninstall and reinstall with every new key issued.

  3. I used to have APSuite version 1.4 or lower and now after installing APSuite 1.5, ASPFront produces and error message and crashes when I try to start the program.

    4. In Windows Explorer, look in the \APSWORK\CONFIG directory. Open the file called APSFRONT.INI. The file will have various lines that reference directories where the APSfront templates and met files are drawn from. Firstly make sure the directories are pointing to the correct relative paths. Secondly, you will notice that the start of the directory reference will have %HOMEDIR or %WORKDIR . For version 1.5 this should now be %APSWORK. The previous versions were are now obsolete and require the newer reference.

  4. I am upgrading from version 1.3. Are there any issues I need to be aware of?

    Yes. In your AUTOEXEC.BAT file in your computer’s root directory, there will be a reference to an \APSIM directory in the SET PATH statement. When you upgrade to version 1.5 there will be two new references in the SET PATH statement. These are \APSUITE\SHARED and \APSUITE\3RDPARTY. To ensure that APSIM runs correctly you must remove the \APSIM reference from the SET PATH statement in the AUTOEXEC.BAT file.
