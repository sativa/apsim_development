Changes to the parameter file

Old (soilwat2.0) parameter files do not need modification. Several parameters are now superfluous, and a new value needs to be added to the ini file:

act_evap_method in the ini file selects which actual soil evaporation is to be used.

Each submodel defined by act_evap_method requires different parameters:
Parameter name
act_evap_method cona u beta max_evap
Ritchie ritchie Yes Yes - -
B&S A bs_a - - Yes -
B&S B bs_b - - Yes -
B&S (andy) bs_acs Yes Yes Yes -
Rickert rickert - - - Yes

  • Andy is using cona and U at initialisation of his model - I'm not sure that he's recycling variable names (heaven forbid!) or assuming incorrectly that they'll always be there.
  • The rickert model needs diffus_xx set to zero, but this interferes with upward flow. A warning is generated if they are non-zero.

A brief description of each new variable:
Cona The change in cumulative second stage evaporation against the square root of time.
U The amount of cumulative evaporation, since soil wetting, before soil supply becomes limiting.
Beta The evaporation characteristic soil parameter determined experimentally.
max_evap The maximum daily soil evaporation (mm).
